# 21. バリデーション設計書

バリデーションはすべてサーバーサイド（Express ルートハンドラ内）で実施。
クライアントサイドのネイティブ HTML5 required 属性も併用するが、サーバー側が正とする。

---

## 1. 共通バリデーション方針

| 項目 | 方針 |
|------|------|
| 実施タイミング | POST/PUT リクエスト受信直後、DB 操作前 |
| エラー形式 | HTML ページ: redirect + query param / API: `{ok:false, error:"..."}` |
| 必須チェック | 空文字・null・undefined すべて NG |
| エスケープ | `escapeHtml()`（lib/helpers.js）で XSS 対策 |
| 文字長制限 | DB 側 maxlength ではなく、ロジック内チェック（一部） |

---

## 2. 認証（auth.js）

| フィールド | ルール | エラーキー |
|-----------|--------|----------|
| username | 必須 | user_not_found |
| password | 必須、bcrypt 照合 | invalid_password |
| 新パスワード | 8文字以上 | password_too_short |
| パスワード確認 | new と一致 | new_password_mismatch |
| 現パスワード | bcrypt 照合 | current_password_wrong |

---

## 3. 勤怠（attendance.js）

| 操作 | フィールド | ルール |
|------|-----------|--------|
| 打刻全般 | userId → Employee | 社員情報が存在すること |
| GPS 打刻 | lat, lng | ApprovedLocation 内の半径以内 |
| 一括登録 | date | 承認済み（isConfirmed=true）の日はスキップ |
| 手動編集 | isConfirmed | true なら 403 |
| 月次承認申請 | — | 申請中の ApprovalRequest が存在しないこと |

---

## 4. 休暇（leave.js）

| フィールド | ルール |
|-----------|--------|
| leaveType | enum 内の値であること |
| startDate / endDate | 必須、startDate <= endDate |
| days | endDate - startDate + 1 で自動計算、0 以下NG |
| leaveBalance | 残日数 >= 申請日数（有給・半休の場合） |
| earlyLeaveTime | 早退の場合は HH:MM 形式 |

---

## 5. 目標（goals.js）

| フィールド | ルール |
|-----------|--------|
| title | 必須、最大 200 文字 |
| deadline | 任意、過去日は警告（ブロックなし） |
| progress | 0〜100 の整数 |
| goalLevel | 低 / 中 / 高 のいずれか |
| status 遷移 | draft のみ submit 可能（他はエラー） |

---

## 6. 掲示板（board.js）

| フィールド | ルール |
|-----------|--------|
| title | 必須 |
| content | 必須 |
| 添付ファイル | 最大 6 件、許可 MIME のみ |

---

## 7. 会社規定（rules.js）

| フィールド | ルール |
|-----------|--------|
| category | 必須 |
| title | 必須 |
| 添付ファイル | 最大 10 件、最大 20MB/件 |

---

## 8. 社員・ユーザー（hr.js / admin.js）

| フィールド | ルール |
|-----------|--------|
| employeeId | 必須、一意（重複チェック） |
| name | 必須 |
| department | 必須 |
| joinDate | 必須、日付形式 |
| email | メール形式（任意項目） |

---

## 9. スキルシート（skillsheet.js）

| フィールド | ルール |
|-----------|--------|
| skills.*.level | 1〜5 の整数 |
| certifications.acquiredDate | 日付形式 |
| projects.periodFrom / periodTo | 日付形式、from <= to |

---

## 10. 残業申請（overtime.js）

| フィールド | ルール |
|-----------|--------|
| date | 必須 |
| startTime / endTime | 必須、HH:MM 形式、start < end |
| reason | 必須 |

## 11. ワークフロー（workflow.js）

| フィールド | ルール |
|-----------|--------|
