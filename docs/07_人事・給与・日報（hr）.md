## 2. ファイルアップロード設定

日報の添付ファイル（社員写真も同じ multer インスタンスを使用）:

| 項目       | 値                                                                    |
| ---------- | --------------------------------------------------------------------- |
| 保存先     | `uploads/daily/`                                                      |
| ファイル名 | `{timestamp}-{random}{ext}`                                           |
| 上限サイズ | 20MB / ファイル                                                       |

| 許可拡張子 | `.jpg/.jpeg/.png/.gif/.webp/.pdf/.docx?/.xlsx?/.pptx?/.txt/.csv/.zip` |

## 4. 給与明細

### 給与明細作成フロー

```
POST /hr/payroll/admin/add
  ├── PayrollRun.create({periodFrom, periodTo, fiscalYear})
  ├── PayrollSlip.create({
  │     runId, employeeId,
  │     workDays, absentDays, lateCount, overtimeHours, nightHours, holidayHours,
  │     baseSalary, allowances[], deductions[], commute, incomeTax,
  │     gross, net, status, notes
  │   })
  └── status が 'issued' 以上 →
        createNotification({ type: 'payslip_issued', 受信者: 該当社員 })
```

### 給与明細更新フロー

```
POST /hr/payroll/admin/edit/:slipId
  ├── PayrollSlip を更新
  └── 旧 status が 'draft' かつ 新 status が 'issued' 以上 →
        createNotification({ type: 'payslip_issued', 受信者: 該当社員 })
```

### ステータス遷移

```
draft → issued → locked → paid
```

| ステータス | 意味                     |
| ---------- | ------------------------ |
| draft      | 作成中・社員未公開       |
| issued     | 発行済み・社員が閲覧可能 |
| locked     | 確定（編集不可）         |
| paid       | 支払済み                 |

### CSV エクスポート（/hr/payroll/:id/export）

- CSV 形式で出力（BOM 付き UTF-8）
- 列: 年, 月, 期間, 基本給, 総支給, 差引支給, ステータス, 所得税, 通勤費（非課税）, 通勤費（課税）, 手当, 控除
- `?payMonth=YYYY-MM` クエリで月別絞り込み可能
- `filename: {氏名}_給与明細_{年}年{月}月.csv`

### 給与明細確認（/hr/payroll/:id/confirm）

```
POST /hr/payroll/:id/confirm
  ├── body: { slipId, confirmedAt }
  ├── PayrollSlip.confirmedAt を更新
  └── 本人または admin のみ許可
```

## 5. 日報

### 日報作成フォームの入力項目

| フィールド   | 必須 | 説明                                                     |
| ------------ | ---- | -------------------------------------------------------- |
| reportDate   | ✅   | 日報日付（Flatpickr ピッカー）                           |
| content      | ✅   | 本文（@メンション対応）                                  |
| achievements | —    | 本日の成果                                               |
| issues       | —    | 課題・問題点                                             |
| tomorrow     | —    | 明日の予定                                               |
| attachments  | —    | 添付ファイル（最大10件・20MB/件・ドラッグ&ドロップ対応） |

**@メンション**: テキスト内で `@名前` を入力するとサジェストが表示され、選択すると `mentions[]` に userId が保存される。投稿時にメンション対象者へ `createNotification({ type: 'mention', ... })` を送信する。

```
POST /hr/daily-report/:id/reaction
  ├── body: { emoji: <上記キーのいずれか> }
  ├── 既に押していれば取り消し（トグル）
  ├── レスポンス: { ok, reacted, count, names }
  └── 自分の日報でなければ →
        createNotification({ type: 'reaction', 受信者: 日報作成者 })
```

### コメント投稿

```
POST /hr/daily-report/:id/comment
  ├── DailyReport の comments 配列に push
  └── 自分の日報でなければ →
        createNotification({ type: 'comment', 受信者: 日報作成者 })
```

### コメントへのスタンプ

```
POST /hr/daily-report/:reportId/comment/:commentId/reaction
  ├── comment.reactions 配列でトグル
  └── 自分のコメントでなければ →
        createNotification({ type: 'reaction', 受信者: コメント投稿者 })
```

### 日報 AI サマリー

- `GET /hr/daily-report/summary` — 週次・月次・社員別の AI 要約ページ
  - クエリ: `?mode=week|month|employee&employeeId=&year=&month=&week=`
  - admin のみ全社員分を閲覧可能。一般ユーザーは自分のみ
- `GET /hr/daily-report/api/stream` — OpenAI API を SSE でストリーミング
  - クエリ: 上記と同じパラメータ
  - `text/event-stream` で chunk を送信
- `POST /hr/daily-report/api/chat` — 日報 AI チャット（非ストリーミング）
  - body: `{ prompt, reportIds }` など
