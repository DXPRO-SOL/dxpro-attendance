## 4. Pay Slips

### Pay Slip Creation Flow

```
POST /hr/payroll/admin/add
  ├── PayrollRun.create({periodFrom, periodTo, fiscalYear})
  ├── PayrollSlip.create({
  │     runId, employeeId,
  │     workDays, absentDays, lateCount, overtimeHours, nightHours, holidayHours,
  │     baseSalary, allowances[], deductions[], commute, incomeTax,
  │     gross, net, status, notes
  │   })
  └── status >= 'issued' →
        createNotification({ type: 'payslip_issued', recipient: employee })
```

### Pay Slip Update Flow

```
POST /hr/payroll/admin/edit/:slipId
  ├── PayrollSlip updated
  └── old status was 'draft' AND new status >= 'issued' →
        createNotification({ type: 'payslip_issued', recipient: employee })
```

### Status Transitions

```
draft → issued → locked → paid
```

| Status | Meaning                              |
| ------ | ------------------------------------ |
| draft  | In progress, not visible to employee |
| issued | Published, employee can view         |
| locked | Finalized, no further edits          |
| paid   | Payment completed                    |

### CSV Export (/hr/payroll/:id/export)

- UTF-8 BOM CSV output
- Columns: Year, Month, Period, Base Salary, Gross, Net, Status, Income Tax, Commute (non-taxable), Commute (taxable), Allowances, Deductions
- `?payMonth=YYYY-MM` query for filtering by month
- `filename: {name}_PaySlip_{year}_{month}.csv`

### Pay Slip Confirmation (/hr/payroll/:id/confirm)

```
POST /hr/payroll/:id/confirm
  ├── body: { slipId, confirmedAt }
  ├── Updates PayrollSlip.confirmedAt
  └── Only allowed for own employee or admin
```

## 5. Daily Reports

### Create Form Fields

| Field        | Required | Description                                       |
| ------------ | -------- | ------------------------------------------------- |
| reportDate   | ✅       | Report date (Flatpickr picker)                    |
| content      | ✅       | Main body (@mention supported)                    |
| achievements | —        | Today's achievements                              |
| issues       | —        | Problems / issues                                 |
| tomorrow     | —        | Tomorrow's plans                                  |
| attachments  | —        | File attachments (max 10, 20MB each, drag & drop) |

**@mention**: Typing `@name` shows a suggestion list; selection saves to `mentions[]`. On submit, sends `createNotification({ type: 'mention', ... })` to mentioned users.

```
POST /hr/daily-report/:id/reaction
  ├── body: { emoji: <one of the keys above> }
  ├── Toggles reaction (removes if already reacted)
  ├── Response: { ok, reacted, count, names }
  └── If not own report →
        createNotification({ type: 'reaction', recipient: report author })
```

```
POST /hr/daily-report/:reportId/comment/:commentId/reaction
  ├── Toggles comment.reactions array
  └── If not own comment →

```

### Daily Report AI Summary

- `GET /hr/daily-report/summary` — Weekly/monthly/per-employee AI summary page
  - Query: `?mode=week|month|employee&employeeId=&year=&month=&week=`
  - Admin can view all employees; regular users see only own reports
- `GET /hr/daily-report/api/stream` — OpenAI API SSE streaming
  - Same query params as above
  - Sends chunks as `text/event-stream`
- `POST /hr/daily-report/api/chat` — AI chat for reports (non-streaming)
  - body: `{ prompt, reportIds }`
