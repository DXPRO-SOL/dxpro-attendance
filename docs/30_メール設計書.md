## 1. メール送信インフラ

### 1-1. 設定ファイル

`config/mailer.js`

### 1-2. 優先順位

```
SENDGRID_API_KEY が SG. で始まる       → @sendgrid/mail を使用
SENDGRID_API_KEY が xkeysib- で始まる  → Brevo REST API（失敗時は SMTP フォールバック）
それ以外                               → SMTP（Brevo SMTP / nodemailer）
```

### 1-3. 環境変数

| 変数名 | 説明 | デフォルト |
|--------|------|----------|
| SENDGRID_API_KEY | SendGrid または Brevo の API キー | — |
| SMTP_HOST | SMTP サーバーホスト | smtp-relay.brevo.com |
| SMTP_PORT | SMTP ポート番号 | 587 |
| SMTP_USER | SMTP 認証ユーザー | — |
| SMTP_PASS | SMTP 認証パスワード | — |
| MAIL_FROM | デフォルト送信元アドレス | no-reply@dxpro-sol.com |

## 2. 送信ヘルパー API

### 2-1. sendMail（config/mailer.js）

```js
sendMail({ to, from, subject, text, html, attachments })
```

低レベル送信関数。from 未指定時は SMTP_USER / MAIL_FROM を使用。

### 2-2. sendEmailToUser（lib/emailHelper.js）

```js
sendEmailToUser(userId, { subject, text, html, from })
// → Promise<boolean>
```

- userId から User を lookup し `email` と `emailVerified` を確認
- 未登録・未認証の場合はスキップして `false` を返す（エラーにしない）
- 例外発生時は console.error のみ（サイレントフェイル）

### 2-3. sendEmailToMultipleUsers（lib/emailHelper.js）

```js
sendEmailToMultipleUsers(userIds, { subject, text, html })
// → Promise<{ success: number, failed: number }>
```

userIds を順次処理（並列なし）。

## 4. 送信仕様・注意事項

| 項目 | 仕様 |
|------|------|
| 非同期実行 | ほぼすべての sendEmailToUser 呼び出しは await なし（fire-and-forget） |
| 失敗時の動作 | メール送信失敗はシステムエラーにならない（console.error のみ） |
| 送信条件 | User.email が存在かつ User.emailVerified === true の場合のみ送信 |
| 件名プレフィックス | 【NOKORI{機能名}】形式で統一 |
| HTML メール | html パラメータ省略時は text がそのまま使用される |
| 添付ファイル | sendMail の attachments パラメータで対応可能（現行利用なし） |
