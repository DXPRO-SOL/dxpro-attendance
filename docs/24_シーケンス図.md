# 24. シーケンス図

主要業務フローのシーケンスを Mermaid sequenceDiagram 形式で示す。

---

## 1. ログインシーケンス

```mermaid
sequenceDiagram
  actor User
  participant Browser
  participant AuthRoute as routes/auth.js
  participant DB as MongoDB
  participant AuditLog

  User->>Browser: username / password 入力
  Browser->>AuthRoute: POST /login
  AuthRoute->>DB: User.findOne({username})
  DB-->>AuthRoute: User or null
  alt ユーザーが存在しない
    AuthRoute->>AuditLog: writeAuditLog(login_failed)
    AuthRoute-->>Browser: redirect /login?error=user_not_found
  else パスワード不一致
    AuthRoute->>AuditLog: writeAuditLog(login_failed)
    AuthRoute-->>Browser: redirect /login?error=invalid_password
  else 認証成功
    AuthRoute->>AuthRoute: req.session.userId = user._id
    AuthRoute->>AuditLog: writeAuditLog(login, success)
    AuthRoute-->>Browser: redirect /dashboard
  end
```

## 2. GPS 打刻シーケンス（出勤）

```mermaid
sequenceDiagram
  actor Employee
  participant Browser
  participant AttRoute as routes/attendance.js
  participant DB as MongoDB

  Employee->>Browser: 出勤ボタンクリック
  Browser->>Browser: navigator.geolocation.getCurrentPosition()
  Browser->>AttRoute: POST /attendance/check-in {lat, lng}
  AttRoute->>DB: ApprovedLocation.find({isActive:true})
  DB-->>AttRoute: 承認場所リスト
  AttRoute->>AttRoute: haversine(userLat, userLng, loc.lat, loc.lng)
  alt 範囲内
    AttRoute->>DB: Attendance.create({checkIn, gpsLat, gpsLng, gpsLocation})
    AttRoute-->>Browser: 200 OK
  else 範囲外
    AttRoute-->>Browser: 400 GPS範囲外エラー
  end
```

## 3. 月次承認シーケンス

```mermaid
sequenceDiagram
  actor Employee
  actor Admin
  participant AttRoute as routes/attendance.js
  participant AdminRoute as routes/admin.js
  participant DB as MongoDB
  participant Email
  participant Notification

  Employee->>AttRoute: POST /attendance/request-approval
  AttRoute->>DB: ApprovalRequest.create({status:pending})
  AttRoute->>Notification: createNotification(admin, attendance_submitted)
  AttRoute-->>Employee: 申請完了

  Admin->>AdminRoute: GET /admin/approval-requests
  AdminRoute->>DB: ApprovalRequest.find({status:pending})
  DB-->>AdminRoute: 申請一覧
  AdminRoute-->>Admin: 一覧表示

  Admin->>AdminRoute: POST /admin/approve/:id
  AdminRoute->>DB: ApprovalRequest.status = approved
  AdminRoute->>DB: Attendance.updateMany({isConfirmed:true})
  AdminRoute->>Email: 承認完了メール送信
  AdminRoute->>Notification: createNotification(employee, attendance_approved)
  AdminRoute-->>Admin: 完了
```

## 4. 目標承認シーケンス（2段階）

```mermaid
sequenceDiagram
  actor Employee
  actor Manager
  actor AdminUser as Admin
  participant GoalRoute as routes/goals.js
  participant DB as MongoDB
  participant Notification

  Employee->>GoalRoute: POST /goals/:id/submit
  GoalRoute->>DB: Goal.status = pending1
  GoalRoute->>Notification: createNotification(manager, goal_submitted)

  Manager->>GoalRoute: POST /goals/:id/approve1
  GoalRoute->>DB: Goal.status = approved1
  GoalRoute->>Notification: createNotification(employee, goal_approved)

  Employee->>GoalRoute: POST /goals/:id/submit2
  GoalRoute->>DB: Goal.status = pending2
  GoalRoute->>Notification: createNotification(admin, goal_submitted2)

  AdminUser->>GoalRoute: POST /goals/:id/approve2
  GoalRoute->>DB: Goal.status = completed
  GoalRoute->>Notification: createNotification(employee, goal_completed)
```

## 5. 休暇承認シーケンス

```mermaid
sequenceDiagram
  actor Employee
  actor Admin
  participant LeaveRoute as routes/leave.js
  participant AdminRoute as routes/admin.js
  participant DB as MongoDB
  participant Email
  participant Slack

  Employee->>LeaveRoute: POST /leave/apply
  LeaveRoute->>DB: LeaveBalance 残日数チェック
  LeaveRoute->>DB: LeaveRequest.create({status:pending})
  LeaveRoute->>DB: Notification.create(admin)
  LeaveRoute->>Email: 申請通知メール

  Admin->>AdminRoute: POST /admin/leave/approve/:id
  AdminRoute->>DB: LeaveRequest.status = approved
  AdminRoute->>DB: LeaveBalance.paid -= days
  AdminRoute->>DB: Attendance 反映（休暇ステータス設定）
  AdminRoute->>Email: 承認通知メール to Employee
  AdminRoute->>Slack: Webhook 通知（設定時）
  AdminRoute->>DB: Notification.create(employee)
```

## 6. 通知配信シーケンス（Socket.IO）

```mermaid
sequenceDiagram
  participant Route as 各ルートファイル
  participant NotifLib as lib/createNotification
  participant DB as MongoDB (Notification)
  participant SocketIO as Socket.IO
  participant Browser

  Route->>NotifLib: createNotification(userId, type, title, body, link)
  NotifLib->>DB: Notification.create({isRead:false})
  NotifLib->>DB: User.findById(userId) → preferredLang
  NotifLib->>NotifLib: localizeNotif(type, lang)
  NotifLib->>SocketIO: io.to(userId).emit("notification", data)
  SocketIO-->>Browser: リアルタイム通知受信
  Browser->>Browser: fetchUnreadCount() バッジ更新
```
