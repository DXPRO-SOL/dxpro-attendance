---

## 1. API 概要

| 項目 | 内容 |
|------|------|
| API 方式 | RESTful（HTTP/HTTPS） |
| データ形式 | JSON（API）/ HTML（ページ） |
| 認証方式 | セッション認証（Cookie ベース） |
| 未認証時 | JSONリクエスト → 401 JSON / ページリクエスト → /loginへリダイレクト |
| HTTPS | Render.com 提供の TLS 終端 |

## 3. 外部システム連携

| 外部システム | 連携目的 | 方式 | 設定 |
|------------|---------|------|------|
| Brevo（旧Sendinblue） | トランザクションメール送信 | SMTP または REST API | SMTP_HOST / SMTP_PORT / SMTP_USER / SMTP_PASS |
| SendGrid | トランザクションメール送信（代替） | REST API | SENDGRID_API_KEY（SG.プレフィックス） |
| MongoDB Atlas | データ永続化 | MongoDB Driver | MONGODB_URI |
| Render.com | アプリホスティング | HTTPS デプロイ | render.yaml |
| Google Fonts | UI フォント | CDN | ネットワーク接続必要 |

## 4. リアルタイム通信

| 項目 | 内容 |
|------|------|
| 方式 | WebSocket（Socket.IO） |
| 用途 | 通知プッシュ・チャット・通話シグナリング |
| 接続管理 | ユーザーIDをキーにルームに参加（u_{userId}） |
| グループ | チャットルームをキーにルーム管理（r_{roomId}） |
| フォールバック | Polling（WebSocket 非対応環境） |

## 5. 認証フロー詳細

| ステップ | 内容 |
|---------|------|
| 1 | POST /login にユーザー名・パスワードを送信 |
| 2 | サーバーが DB でユーザーを検索し bcrypt でパスワード照合 |
| 3 | 照合成功: セッションに userId / isAdmin / orgRole / lang を保存 |
| 4 | 照合失敗: /login?error=キー にリダイレクト、監査ログに記録 |
| 5 | 以降のリクエスト: Cookie のセッション ID で認証状態を維持 |
| 6 | ログアウト: GET /logout でセッション破棄 |
