# 認証 - 機能設計書

---

## 1. 機能概要
ユーザーのログイン認証・セッション管理・パスワード変更・ログアウトを管理する機能。
bcrypt によるパスワードハッシュ化とセッション Cookie で認証状態を維持する。

---

## 2. 機能一覧
| 機能ID | 機能名 | 利用者 |
|--------|--------|--------|
| AUTH-01 | ログイン | 全ユーザー |
| AUTH-02 | ログアウト | ログイン済みユーザー |
| AUTH-03 | パスワード変更 | ログイン済みユーザー |
| AUTH-04 | セッション管理 | 自動 |
| AUTH-05 | 監査ログ記録 | 自動 |

---

## 3. 処理概要

### AUTH-01 ログイン
1. POST /login にユーザー名・パスワードを送信
2. DB でユーザーを検索（username で照合）
3. bcrypt.compare でパスワードを検証
4. 成功: セッションに userId / isAdmin / orgRole / lang を保存
5. 失敗: /login?error=キー にリダイレクト
6. 成功・失敗ともに AuditLog に記録

### AUTH-04 セッション管理
- express-session + MongoDB セッションストアで管理
- セッション有効期限: 環境設定による（デフォルト 24h）
- 未認証アクセス: JSON リクエスト → 401 / ページ → /login リダイレクト
