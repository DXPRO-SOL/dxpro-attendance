# BD-11. 権限設計書

---

## 1. ロール一覧

| ロール | 対象者 | 主な権限 |
|--------|--------|---------|
| admin（管理者） | システム管理担当・人事担当 | 全機能フルアクセス。社員登録・給与発行・全承認操作・設定変更 |
| manager（マネージャー） | 部門長・課長相当 | 部門の勤怠確認・目標1次承認・チームスケジュール管理 |
| team_leader（チームリーダー） | チームリーダー | チームの勤怠確認・スケジュール管理 |
| employee（一般社員） | 一般社員 | 自分のデータ登録・閲覧・各種申請 |
| test_user（テストユーザー） | デモ・テスト用アカウント | 全機能の閲覧のみ可（書き込み操作はすべてブロック） |

## 3. データアクセス制御

| データ種別 | 自分のデータ | 他人のデータ |
|-----------|------------|------------|
| 勤怠記録 | 閲覧・編集 | 管理者のみ閲覧・編集 |
| 給与明細 | 閲覧 | 管理者のみ閲覧・編集 |
| 目標 | 閲覧・編集 | 承認者・管理者は閲覧 |
| スキルシート | 閲覧・編集 | 管理者は閲覧 |
| 休暇申請 | 閲覧・申請 | 管理者のみ閲覧・承認 |
| チャット | 自室のみ | — |
| クラウドファイル | 共有設定による | 権限者のみ |

---

## 4. 特殊制御

| 制御 | 対象 | 内容 |
|------|------|------|
| 確認済み勤怠の変更禁止 | 全ロール | 管理者承認後の勤怠は社員による修正不可 |
| テストユーザー書き込み禁止 | test_user | POST/PUT/DELETE 操作はすべて403エラー |
| 未認証アクセス | 全ロール | ログインが必要な画面は/loginへリダイレクト |
| 入社前テスト | 認証不要 | /pretestは認証なしでアクセス可能 |
