# 目標管理 - 画面設計書

---

## 1. 目標一覧（/goals）

### 概要
自分の目標一覧を表示。ステータスでフィルタリングできる。

### 画面レイアウト
| ゾーン | 表示内容 |
|--------|---------|
| フィルター | ステータス（全て/下書き/承認待ち/完了等） |
| 目標カード | タイトル・重要度・期限・進捗率・ステータスバッジ |
| 新規作成ボタン | 目標作成フォームへ遷移 |
| 承認タブ（承認者） | 自分が承認者の承認待ち目標一覧 |

---

## 2. 目標作成・編集フォーム（/goals/add, /goals/edit/:id）

### 項目一覧
| 項目名 | 種別 | 必須 | 説明 |
|--------|------|------|------|
| タイトル | テキスト | 必須 | 目標名称 |
| 説明 | テキストエリア | 任意 | 目標の詳細内容 |
| 重要度 | 選択 | 任意 | 低/中/高 |
| 期限 | 日付 | 任意 | 達成期限 |
| アクションプラン | テキストエリア | 任意 | 具体的な行動計画 |
| AI提案 | ボタン | — | AIが目標内容のサジェストを提案 |

### ボタン動作
| ボタン | 動作 |
|--------|------|
| 下書き保存 | status: draft で保存 |
| 1次承認申請 | status: pending1 に変更、承認者へ通知 |

---

## 3. 目標詳細（/goals/detail/:id）

### 画面レイアウト
| ゾーン | 表示内容 |
|--------|---------|
| 基本情報 | タイトル・説明・重要度・期限・アクションプラン |
| 進捗 | 進捗率バー（現在値）、評価グレード（承認後） |
| ステータス | 現在のステータスバッジ |
| 承認ボタン（1次） | マネージャーのみ表示（pending1時） |
| 評価・2次申請 | 作成者のみ表示（approved1時） |
| 最終承認ボタン | 管理者のみ表示（pending2時） |
| 差し戻しボタン | 承認者のみ表示 |
| 履歴 | 承認・差し戻しの操作履歴一覧 |

---

## 4. 評価入力フォーム（/goals/evaluate/:id）

### 項目一覧
| 項目名 | 種別 | 必須 | 説明 |
|--------|------|------|------|
| 進捗率 | スライダー | 必須 | 0〜100% |
| 評価グレード | 選択 | 任意 | A/B/C/S/D等（管理者が設定） |
| 最終コメント | テキストエリア | 任意 | 自己評価コメント |

---

## 5. 承認待ち一覧（/goals/approval）

### 概要
自分が承認担当者として割り当てられた目標の一覧。

### 画面レイアウト
| ゾーン | 表示内容 |
|--------|---------|
| 承認待ちテーブル | 申請者名・タイトル・申請日・ステータス |
| 操作ボタン | 詳細表示→承認/差し戻し |

---

## 6. 管理者: 目標レポート（/goals/report）

### 画面レイアウト
| ゾーン | 表示内容 |
|--------|---------|
| サマリー | ステータス別集計、承認率 |
| 社員一覧 | 社員名・目標数・完了数・未提出数 |
| フィルター | 部署・ステータス・期間 |
