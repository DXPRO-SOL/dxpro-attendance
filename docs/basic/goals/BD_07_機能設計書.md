# 目標管理 - 機能設計書

---

## 1. 機能概要
社員が個人目標を設定し、マネージャーによる1次承認・管理者による2次承認を経て確定する
2段階承認フローの目標管理（MBO）機能。

---

## 2. 機能一覧
| 機能ID | 機能名 | 利用者 |
|--------|--------|--------|
| GOA-01 | 目標作成（下書き） | 社員 |
| GOA-02 | 目標編集 | 社員 |
| GOA-03 | 1次承認申請 | 社員 |
| GOA-04 | 1次承認 | マネージャー |
| GOA-05 | 1次差し戻し | マネージャー |
| GOA-06 | 評価入力・2次申請 | 社員 |
| GOA-07 | 2次承認（最終） | 管理者 |
| GOA-08 | 2次差し戻し | 管理者 |
| GOA-09 | 進捗更新 | 社員 |
| GOA-10 | 目標削除（下書きのみ） | 社員 |
| GOA-11 | AI目標提案 | 社員 |
| GOA-12 | 目標レポート | 管理者 |

---

## 3. 処理概要

### GOA-03 1次承認申請
1. 社員が「1次承認申請」操作
2. status: draft → pending1
3. 承認担当者（approver1）を決定（マネージャー/管理者）
4. 承認者へ通知・メール送信
5. 履歴（history）に操作を記録

### GOA-07 最終承認
1. 管理者が「最終承認」操作
2. 評価グレードを設定（任意）
3. status: pending2 → completed
4. 作成者へ通知・メール送信
5. 履歴に記録

### GOA-11 AI目標提案
1. GET /api/ai/goal-suggestions を呼び出し
2. AIがシステム内のデータを参照して目標のサジェストを返却
3. フォームに反映（編集可）
