# 通知システム - 機能設計書

## 1. 機能概要
業務イベント（承認・却下・発行等）が発生した際に、関係者へリアルタイム通知と
メール通知を送信する機能。Socket.IO でブラウザへプッシュ配信する。

## 2. 機能一覧
| 機能ID | 機能名 | 利用者 |
|--------|--------|--------|
| NTF-01 | リアルタイム通知受信 | 全員 |
| NTF-02 | 通知一覧閲覧 | 全員 |
| NTF-03 | 個別既読 | 全員 |
| NTF-04 | 一括既読 | 全員 |
| NTF-05 | 未読バッジ更新（リアルタイム） | 全員 |

## 3. 処理概要

### NTF-01 通知送信
1. 業務イベント発生（承認・却下等）
2. createNotification() で Notification レコードを DB に作成
3. io.to("u_{userId}").emit("notification", data) で Socket.IO 送信
4. ブラウザ側: ベルバッジの未読数を即時更新

### NTF-05 バッジリアルタイム更新
- Socket.IO 接続確立時にユーザー固有ルーム（u_{userId}）に参加
- 新規通知受信時にバッジカウントをインクリメント
