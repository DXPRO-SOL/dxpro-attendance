# ワークフロー - 画面設計書

---

## 1. ワークフロー一覧（/workflow）SPA画面

### 概要
申請の作成・一覧確認・承認操作をすべて1画面（SPA）で行う。詳細はモーダルで表示。

### 画面レイアウト
| ゾーン | 表示内容 |
|--------|---------|
| タブ | 自分の申請 / 承認待ち（自分が承認者） / 全申請（管理者） |
| フィルター | フォーム種別・ステータス・キーワード |
| 申請一覧テーブル | シリアルNO・フォーム名・件名・申請者・申請日・ステータス |
| 新規申請ボタン | フォーム選択モーダルを開く |
| 詳細ボタン | 詳細モーダルを開く |

### 詳細モーダル
| ゾーン | 表示内容 |
|--------|---------|
| 申請内容 | フォーム項目と入力値の一覧表示 |
| 承認経路 | 承認ステップ一覧（承認者・ステータス・処理日時） |
| コメント | 申請に関するコメントスレッド |
| 操作ボタン | 承認/差し戻し/却下（承認者のみ） |

---

## 2. フォーム定義管理（管理者）

### 概要
管理者が申請フォームの項目と承認フローを定義する。

### フォーム項目定義
| 項目名 | 説明 |
|--------|------|
| フォーム名 | 申請種別の名称（例: 経費申請、休暇申請） |
| フィールド定義 | テキスト/数値/日付/選択肢/テキストエリアの項目を追加 |
| 承認フロー | 承認ステップを順序付きで設定（承認者: ユーザー/ロール/管理者等） |
| 承認タイプ | 全員承認 / 1名承認（グループ承認） |
