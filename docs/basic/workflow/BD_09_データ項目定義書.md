# ワークフロー - データ項目定義書

---

## 1. ワークフロー申請（Workflow）
| 項目名 | 説明 | 型 | 必須 |
|--------|------|-----|------|
| serialNumber | 申請番号（採番） | 文字列 | 自動 |
| formId | 使用したフォーム定義ID | 参照 | 必須 |
| title | 申請件名 | 文字列 | 必須 |
| applicantId | 申請者ユーザーID | 参照 | 必須 |
| fields | フォーム入力値（key-value） | オブジェクト | 必須 |
| status | 申請ステータス | 区分 | 自動 |
| steps | 承認ステップ配列 | 配列 | 自動 |
| history | 操作履歴配列 | 配列 | 自動 |
| comments | コメント配列 | 配列 | 自動 |

### status コード値
| 値 | 説明 |
|----|------|
| draft | 下書き |
| submitted | 申請中 |
| approved | 承認完了 |
| returned | 差し戻し |
| rejected | 却下 |

### steps 配列要素
| 項目名 | 説明 |
|--------|------|
| order | ステップ順序（1始まり） |
| name | ステップ名 |
| assigneeType | 承認者種別（user/role/manager等） |
| assignees | 承認担当者ID配列 |
| approvalType | all（全員承認）/ any（1名承認） |
| status | ステップのステータス（pending/approved/returned/rejected/skipped） |
| processedBy | 処理した承認者ID |
| processedAt | 処理日時 |
| comment | 承認コメント |

### history 配列要素
| 項目名 | 説明 |
|--------|------|
| action | 操作（created/submitted/approved/returned/rejected/resubmitted/delegated/commented） |
| userId | 操作者ID |
| comment | コメント |
| timestamp | 操作日時 |
