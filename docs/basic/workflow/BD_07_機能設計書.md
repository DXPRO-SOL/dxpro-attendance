# ワークフロー - 機能設計書

---

## 1. 機能概要
管理者が定義した任意の申請フォームを使い、多段階承認の電子申請を行う汎用ワークフロー機能。
申請ごとにシリアルナンバーが採番され、承認履歴を時系列で追跡できる。

---

| 機能ID | 機能名 | 利用者 |
|--------|--------|--------|
| WF-01 | 申請作成・送信 | 全社員 |
| WF-02 | 申請一覧・検索 | 全社員 |

| WF-08 | フォーム定義作成 | 管理者 |
| WF-09 | 承認フロー定義 | 管理者 |
| WF-10 | レポート出力 | 管理者 |

---

## 3. 処理概要

### WF-01 申請作成・送信
1. 申請者がフォームを選択
2. フォーム項目に入力
3. POST /api/workflow で申請送信
4. シリアルナンバー自動採番（フォーム名-YYYYMM-連番）
5. 最初の承認ステップの承認者へ通知・メール送信

### WF-04 承認操作
1. 承認者が承認操作（/api/workflow/:id/approve）
2. 現在ステップを完了としてマーク
3. 次ステップがあれば次承認者へ通知
4. 全ステップ完了で申請ステータス → approved
5. 申請者へ完了通知・メール送信
6. 履歴に操作を記録
