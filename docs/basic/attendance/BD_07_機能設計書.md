# 勤怠管理 - 機能設計書

---

## 1. 機能概要
社員が日々の出退勤を打刻・記録し、月次で管理者承認を経て確定するまでの勤怠管理機能。
GPS による位置情報検証で適切な場所からの打刻のみを受け付ける。

## 3. 処理概要

### ATT-01/03 打刻（GPS）
1. ブラウザ側: `navigator.geolocation` で現在位置取得
2. `/locations/api/active` で有効な承認場所を取得
3. ハバーサイン距離計算で半径内かどうかを判定
4. 圏内の場合: GPS座標を付与してサーバーへ送信
5. サーバー側でも再度GPS検証（二重検証）
6. 承認後、Attendance レコードを作成・更新

### ATT-07 月次承認申請
1. 社員が /my-monthly-attendance で内容確認
2. 「承認申請」ボタン → POST /request-approval
3. ApprovalRequest レコードを作成（status: pending）
4. 管理者へ通知・メール送信
5. 申請済み月は社員による修正不可

### ATT-08 月次勤怠承認
1. 管理者が承認操作 → 対象月全勤怠を `confirmed: true` に変更
2. ApprovalRequest.status → approved
3. 社員へ通知・メール送信

## 4. 自動計算ロジック
| 項目 | 計算式 |
|------|--------|
| 昼休憩時間 | lunchEnd - lunchStart（打刻がある場合のみ） |
| 実労働時間 | (退勤 - 出勤) - 昼休憩時間（小数点1位） |
| 残業時間 | max(0, 実労働時間 - 8時間) |
| ステータス自動判定 | 退勤時: 実労働時間 < 8 → 早退 |
