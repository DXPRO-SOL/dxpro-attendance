# 休暇申請 - 機能設計書

---

## 1. 機能概要
社員が休暇申請を行い管理者が承認・却下する機能。有給・病欠等の残日数を自動管理し、
承認時には勤怠ステータスへ自動反映する。

---

## 2. 機能一覧
| 機能ID | 機能名 | 利用者 |
|--------|--------|--------|
| LEV-01 | 休暇申請 | 社員 |
| LEV-02 | 早退申請 | 社員 |
| LEV-03 | 申請一覧表示 | 社員 |
| LEV-04 | 申請取消 | 社員 |
| LEV-05 | 休暇承認 | 管理者 |
| LEV-06 | 休暇却下 | 管理者 |
| LEV-07 | 残日数一覧 | 管理者 |
| LEV-08 | 残日数付与・調整 | 管理者 |

---

## 3. 処理概要

### LEV-01 休暇申請
1. 申請フォーム入力
2. 残日数チェック（申請日数 ≤ 残日数）
3. LeaveRequest レコード作成（status: pending）
4. 管理者へ通知・メール送信

### LEV-05 休暇承認
1. 管理者が承認操作
2. LeaveRequest.status → approved
3. LeaveBalance から残日数を差し引き（LeaveBalance.history に記録）
4. 対象日の勤怠レコードを自動更新:
   - 有給 → status: 有休、workingHours: 8
   - 病欠 → status: 欠勤、workingHours: 0
   - 慶弔/その他 → status: 休暇、workingHours: 0
   - 午前休/午後休/早退 → status: 各区分、workingHours: 4
5. 申請者へ通知・メール送信

### LEV-06 休暇却下
1. 管理者が却下操作（却下理由入力）
2. LeaveRequest.status → rejected
3. 残日数は変更しない
4. 申請者へ通知・メール送信
