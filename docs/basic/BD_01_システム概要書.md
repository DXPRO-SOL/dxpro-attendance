# BD-01. システム概要書

---

## 1. システム目的

本システムは、中小規模企業（主に外国籍社員を含む職場）を対象とした**勤怠・人事・HR 統合管理システム**である。
以下を目的とする：

- 勤怠記録・月次集計・承認フローの完全デジタル化
- 多言語対応（5言語）による外国籍社員の利便性向上
- 目標管理・給与・スキル管理の一元化による HR 業務 DX 推進
- リアルタイム通知・チャットによる社内コミュニケーション効率化

---

## 2. システム概要

| 項目 | 内容 |
|------|------|
| システム名 | NOKORI 勤怠・HR 管理システム |
| 提供形態 | Web アプリケーション（SaaS） |
| 対応言語 | 日本語 / 英語 / ベトナム語 / 韓国語 / 中国語 |
| 利用端末 | PC ブラウザ・スマートフォン（レスポンシブ対応） |
| 推奨ブラウザ | Google Chrome / Safari 最新版 |
| データ保存 | クラウド（MongoDB Atlas） |

---

## 3. システム構成

| コンポーネント | 役割 | 採用技術 |
|--------------|------|---------|
| アプリケーションサーバー | 画面・API 提供 | Node.js / Express（Render.com） |
| データベース | 全データ永続化 | MongoDB Atlas |
| リアルタイム通信 | 通知・チャット・通話 | Socket.IO |
| メール配信 | 各種通知メール送信 | Brevo（SMTP/REST）/ SendGrid |
| ファイル保管 | 添付ファイル・帳票 | サーバーローカル（uploads/） |

---

## 4. 対象業務

| 業務分類 | 機能 |
|---------|------|
| 勤怠管理 | 出退勤打刻（GPS対応）・月次集計・承認申請 |
| 休暇管理 | 休暇申請・残高管理・承認フロー |
| 目標管理（MBO） | 個人目標設定・進捗管理・2段階承認 |
| 給与管理 | 給与明細発行・CSV出力 |
| 日報管理 | 日次報告・週次/月次 AI 要約 |
| スキルシート | スキル登録・Excel エクスポート |
| 掲示板 | 社内お知らせ投稿・閲覧 |
| チャット | リアルタイムグループチャット・DM・音声通話 |
| ワークフロー | 汎用電子申請フロー |
| スケジュール | 社内カレンダー管理 |
| 会社規定 | 規定文書管理・閲覧 |
| 入社前テスト | 採用候補者向けオンラインテスト（認証不要） |
| 契約管理 | 雇用契約書管理 |
| クラウドストレージ | 社内ファイル共有 |
| 通知 | システム横断リアルタイム通知 |
| AI チャットボット | 社内 FAQ・業務アシスタント |

---

## 5. 利用者

| 利用者区分 | ロール | 主な利用業務 |
|-----------|--------|------------|
| 管理者 | admin | 全機能・全社員データ管理・承認・給与発行 |
| マネージャー | manager | 部門管理・目標1次承認・部門勤怠確認 |
| チームリーダー | team_leader | チーム勤怠確認・スケジュール管理 |
| 一般社員 | employee | 勤怠打刻・各種申請・目標設定 |
| テストユーザー | test_user | 閲覧専用（書き込み不可） |
| 入社前候補者 | — | 入社前テストのみ（ログイン不要） |

---

## 6. 利用環境

| 項目 | 条件 |
|------|------|
| ブラウザ | Chrome 最新 / Safari 最新（Internet Explorer 非対応） |
| スマートフォン | iOS Safari / Android Chrome（GPS打刻機能使用時は位置情報許可必要） |
| ネットワーク | インターネット接続必須 |
| 認証方式 | ユーザー名＋パスワード（セッション認証） |
| セッション | サーバーサイドセッション（ブラウザ閉じると終了） |
