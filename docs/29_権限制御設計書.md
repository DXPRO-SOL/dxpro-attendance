# 29. 権限制御設計書

定義ファイル: `middleware/auth.js`

---

## 1. ロール体系

| ロール | orgRole 値 | レベル | 説明 |
|--------|-----------|--------|------|
| 管理者 | admin | 4 | 全機能アクセス可能 |
| マネージャー | manager | 3 | 部門管理・承認操作 |
| チームリーダー | team_leader | 2 | チーム管理操作 |
| 一般社員 | employee | 1 | 自分のデータのみ |
| テストユーザー | — | — | 閲覧専用（書き込みブロック） |

ロールは `User.isAdmin`（boolean）と `Employee.orgRole`（string）の2系統で管理される。
セッションに `req.session.isAdmin` と `req.session.orgRole` として保持。

## 2. ミドルウェア一覧

### 2-1. requireLogin

```js
function requireLogin(req, res, next)
```

| 条件 | 動作 |
|------|------|
| セッションなし + JSON リクエスト | 401 JSON `{ error: "認証が必要です" }` |
| セッションなし + ページリクエスト | redirect /login |
| セッションあり | next() |

---

### 2-2. isAdmin

```js
function isAdmin(req, res, next)
```

| 条件 | 動作 |
|------|------|
| session.isAdmin === true | next() |
| それ以外 + JSON | 403 JSON `{ error: "管理者権限が必要です" }` |
| それ以外 + ページ | 403 テキスト |

---

### 2-3. isManagerOrAdmin

```js
function isManagerOrAdmin(req, res, next)
```

| 条件 | 動作 |
|------|------|
| isAdmin または orgRole が admin/manager | next() |
| それ以外 + JSON | 403 JSON `{ error: "部門長以上の権限が必要です" }` |
| それ以外 + ページ | 403 テキスト |

---

### 2-4. isLeaderOrAbove

```js
function isLeaderOrAbove(req, res, next)
```

| 条件 | 動作 |
|------|------|
| ROLE_LEVEL[orgRole] >= 2（team_leader 以上） | next() |
| それ以外 | 403 |

---

### 2-5. requireRole(...roles)

```js
// ファクトリ関数 → ミドルウェア関数を返す
function requireRole(...roles)

// 使用例
router.post('/path', requireLogin, requireRole('admin','manager'), handler)
```

複数ロールを許可する汎用ガード。ROLE_LEVEL によるレベル比較も使用。

---

### 2-6. blockTestUser

```js
function blockTestUser(req, res, next)
```

| 条件 | 動作 |
|------|------|
| session.isTestUser === true | 403 JSON `{ error: "テストユーザーはこの操作を実行できません" }` |
| それ以外 | next() |

書き込み系 POST/PUT/DELETE ルートに付加してテストユーザーの破壊的操作を防ぐ。

---

### 2-7. wantsJson（内部ユーティリティ）

```js
function wantsJson(req)  // → boolean
```

| 判定条件 | true になる場合 |
|---------|--------------|
| パス | /api/ で始まる |
| req.xhr | true |
| X-Requested-With | XMLHttpRequest |
| Accept | application/json を含む |

エラーレスポンス形式（リダイレクト vs JSON）の分岐に使用。

## 4. データアクセス制御（行レベル）

| モデル | 制御方法 |
|--------|---------|
| Attendance | userId === req.session.userId または isAdmin |
| Goal | createdBy === req.session.userId または currentApprover または isAdmin |
| LeaveRequest | userId === req.session.userId または isAdmin |
| PayrollSlip | employee.userId === req.session.userId または isAdmin |
| DailyReport | userId === req.session.userId（hr/daily-report） |
| SkillSheet | employeeId の userId === req.session.userId または isAdmin |
| CloudFile | permissions[] に userId が含まれる または isOwner または isAdmin |
