# 28. 画面詳細設計書

各画面の URL・認証要件・主要要素・アクションを記載する。

---

## 1. ログイン（/login）

| 項目 | 内容 |
|------|------|
| 認証 | 不要 |
| レイアウト | 共通レイアウトなし（独自デザイン） |
| 主要要素 | 言語セレクター（5言語）、ロゴ、時計、ログインフォーム |
| フォーム | username（必須）、password（必須）、パスワード表示トグル |
| アクション | POST /login |
| エラー表示 | ?error= クエリでサーバーサイドメッセージ表示 |

---

## 2. 勤怠管理トップ（/attendance-main）

| 項目 | 内容 |
|------|------|
| 認証 | requireLogin |
| レイアウト | 独自（共通サイドバーなし） |
| 主要要素 | KPI カード（月間勤怠）、今日の打刻エリア、月次勤怠テーブル |
| KPI | 出勤日数 / 総労働時間 / 残業時間 / 欠勤日数 |
| 打刻ボタン | 出勤 / 退勤 / 昼休憩開始 / 昼休憩終了（状態により制御） |
| 月次テーブル | 日付・出退勤時刻・実労働時間・残業・ステータス・備考 |
| アクション | POST /attendance/check-in / check-out / lunch-start / lunch-end |

---

## 3. ダッシュボード（/dashboard）

| 項目 | 内容 |
|------|------|
| 認証 | requireLogin |
| レイアウト | 共通レイアウト（サイドバー + トップバー） |
| 主要ウィジェット | 今日の勤怠状況、今月の労働時間、休暇残日数、目標進捗、給与明細（最新）、チーム勤怠状況、AI推奨事項、半期評価グレード、未読通知 |
| AI推奨 | computeAIRecommendations() の結果（最大 9 ルール） |
| 半期評価 | computeSemiAnnualGrade() の S/A/B/C/D 表示 |

## 4. 目標一覧（/goals）

| 項目 | 内容 |
|------|------|
| 認証 | requireLogin |
| 表示切替 | 自分の目標 / チームの目標（マネージャー）/ 全員（管理者） |
| フィルター | ステータス別（draft / pending / approved / completed） |
| カード要素 | タイトル、進捗バー、ステータスバッジ、期日、担当者 |
| アクション | 新規作成ボタン → /goals/new |

---

## 5. 目標詳細（/goals/:id）

| 項目 | 内容 |
|------|------|
| 認証 | requireLogin |
| 主要要素 | タイトル、説明、進捗バー（編集可）、ステータス、承認履歴タイムライン |
| アクション（本人） | 提出・進捗更新・編集（draft 時） |
| アクション（マネージャー） | 承認1 / 否認1 |
| アクション（管理者） | 承認2 / 否認2 / グレード設定 |
