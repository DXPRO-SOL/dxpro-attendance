## 2. 主要遷移フロー

### 2-1. ログインフロー

```
GET /login
  → 認証成功 → GET /dashboard
  → 認証失敗 → GET /login?error=invalid_password
  → 未ログインで任意ページ → GET /login
```

### 2-2. 勤怠打刻フロー

```
GET /attendance-main
  → [出勤] POST /attendance/check-in → リロード /attendance-main
  → [退勤] POST /attendance/check-out → リロード /attendance-main
  → [月次表示] → GET /attendance/monthly
  → [月次承認申請] POST /attendance/request-approval → /attendance/monthly
```

### 2-3. 月次承認フロー（管理者）

```
GET /admin/approval-requests
  → [承認] POST /admin/approve/:id → /admin/approval-requests
  → [差し戻し] POST /admin/return/:id → /admin/approval-requests
  → [詳細確認] GET /admin/attendance/:userId/:year/:month
  → [PDF出力] GET /admin/export-pdf/:userId/:year/:month
```

### 2-4. 目標ワークフロー

```
GET /goals → GET /goals/new → POST /goals (draft)
  → [提出] POST /goals/:id/submit (→pending1)
  → [承認1] POST /goals/:id/approve1 (→approved1)
  → [提出2] POST /goals/:id/submit2 (→pending2)
  → [承認2] POST /goals/:id/approve2 (→completed)
```

### 2-5. 休暇申請フロー

```
GET /leave/apply → POST /leave/apply (pending)
  → [管理者] GET /admin/leave-requests
  → [承認] POST /admin/leave/approve/:id
  → [申請者への通知] → /leave/history
```

### 2-6. ログアウト

```
GET /logout → セッション破棄 → GET /login
```
