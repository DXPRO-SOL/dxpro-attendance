# 25. ログ設計書

---

## 1. ログの種類

| ログ種別 | 記録先 | 目的 |
|---------|--------|------|
| 監査ログ（AuditLog） | MongoDB: auditlogs コレクション | 操作者・内容の追跡 |
| アプリケーションログ | サーバー標準出力（console.log/error） | デバッグ・障害調査 |
| アクセスログ | 未実装（Morgan 等なし） | — |

---

## 2. 監査ログ（AuditLog）仕様

### 書き込み関数

```js
// lib/auditLog.js
writeAuditLog(req, {
  action,       // 必須: 操作種別
  category,     // 任意: カテゴリ（デフォルト: ""）
  targetId,     // 任意: 操作対象 ID
  targetModel,  // 任意: 操作対象モデル名
  detail,       // 任意: 詳細説明
  result,       // 任意: success / failure（デフォルト: "success"）
  username,     // セッション外でユーザー名を指定する場合
  userId,       // セッション外でユーザー ID を指定する場合
})
```

### スキーマ（models/index.js の AuditLog）

| フィールド | 型 | 説明 |
|-----------|-----|------|
| userId | ObjectId | 操作者（null = 未認証） |
| username | String | 操作者ユーザー名 |
| action | String | 操作種別 |
| category | String | カテゴリ |
| targetId | String | 操作対象 ID |
| targetModel | String | 操作対象モデル名 |
| detail | String | 詳細説明 |
| ipAddress | String | IP（x-forwarded-for 対応） |
| userAgent | String | ブラウザ UA |
| result | String | success / failure |
| createdAt | Date | 記録日時（自動） |

## 3. action 一覧

| action | 説明 | 記録箇所 |
|--------|------|---------|
| login | ログイン成功 | routes/auth.js |

## 5. IP 取得ロジック

```js
const ipAddress =
  (req.headers["x-forwarded-for"] || "").split(",")[0].trim() ||
  req.socket?.remoteAddress ||
  req.ip ||
  "";
```

- プロキシ・ロードバランサー経由の場合は X-Forwarded-For の先頭を使用

---

## 6. エラー処理

- `writeAuditLog` 内の例外は `console.error` で出力されるが、呼び出し元に例外を投げない
- 監査ログの書き込み失敗はシステム動作を止めない（サイレントフェイル）

---

## 7. 閲覧・エクスポート

- 管理者向け: `GET /auditlog`（routes/auditlog.js）
- 検索条件: action / category / userId / 期間
- CSV エクスポート: `GET /auditlog/export-csv`
