# 22. エラー設計書

---

## 1. エラー応答の基本方針

| リクエスト種別 | 判定方法 | 応答形式 |
|--------------|---------|---------|
| JSON/XHR リクエスト | `wantsJson(req)`（Accept: application/json / X-Requested-With） | `res.json({ok:false, error:"..."})` |
| 通常ページリクエスト | 上記以外 | `res.redirect("?error=key")` または `res.send(HTML)` |
| 認証未通過 | — | JSON: 401 / ページ: /login へリダイレクト |
| 権限不足 | — | JSON: 403 / ページ: /dashboard へリダイレクト |

---

## 2. HTTP ステータスコード一覧

| コード | 意味 | 主な使用箇所 |
|--------|------|------------|
| 400 | Bad Request（必須項目欠損・業務ルール違反） | 全ルート |
| 401 | Unauthorized（未ログイン） | requireLogin ミドルウェア |
| 403 | Forbidden（権限不足・承認済みレコード操作） | isAdmin / blockTestUser / 業務ルール |
| 404 | Not Found（リソース不存在） | 全 findById / findOne |
| 500 | Internal Server Error（例外キャッチ） | 全 try-catch catch ブロック |

---

## 3. エラーキー一覧（?error= / JSON error フィールド）

### 認証系

| キー | 説明 |
|------|------|
| user_not_found | ユーザーが存在しない |
| invalid_password | パスワード不一致 |
| current_password_wrong | 現在のパスワードが違う |
| new_password_mismatch | 新パスワードと確認が不一致 |
| password_too_short | 新パスワードが 8 文字未満 |
| server_error | サーバー内部エラー |

### 業務系（共通）

| キー / メッセージ | 説明 |
|---------|------|
| 社員情報がありません | Employee レコード未作成 |
| 承認済みの勤怠記録は編集できません | isConfirmed=true のレコード |
| アクセス拒否 | 403 権限なし |
| ファイルが見つかりません | 404 |
| コンテンツを読み込めません | ファイル読み取り失敗 |

## 4. エラー処理パターン

### パターン A: try-catch + 500 HTML
```js
try {
  // DB操作
} catch (err) {
  res.status(500).send("○○中にエラーが発生しました");
}
```
主に: ページレンダリング系ルート

### パターン B: try-catch + 500 JSON
```js
try {
  // DB操作
} catch (err) {
  res.status(500).json({ ok: false, error: err.message });
}
```
主に: API エンドポイント（AJAX）

### パターン C: 業務バリデーション + redirect
```js
if (!employee) return res.status(400).send("社員情報がありません");
// または
return res.redirect("/change-password?error=password_too_short");
```
主に: フォーム POST 後

### パターン D: wantsJson 分岐（requireLogin ミドルウェア）
```js
if (wantsJson(req)) {
  return res.status(401).json({ error: "unauthorized" });
}
res.redirect("/login");
```

## 5. 監査ログとエラーの関係

- 認証失敗（login_failed）は AuditLog に記録される
- writeAuditLog 内のエラーは console.error で出力されるが、システムを止めない（サイレントフェイル）
- 500 エラー時は AuditLog には記録されない（キャッチ外のため）

## 6. フロントエンドエラー表示

| 表示方法 | 使用箇所 |
|---------|---------|
| ?error= クエリ + サーバーサイドでメッセージレンダリング | ログイン、パスワード変更 |
| Toast 通知（JS） | AJAX 操作の成功・失敗 |
| インライン HTML エラーメッセージ | 400 応答のページ埋め込み |
